### 2.   Einstellungen Blockly

-   `door_states` Hier müssen nun die States, die überwacht werden sollen, hinzugefügt und bennannt werden. Diese Objekte werden dann unter alias.0.<alias_object_name> angelegt. Müssen Objekt gewechselt oder gelöscht werden, dann bitte diese Änderung im Blockly ausführen. Objekte die hier nicht eingefügt sind, werden um 14 Uhr automatisch überprüft und können gelöscht werden. Diese Überprüfung kann weiter unten eingestellt werden. Die States dürfen nur 0, 1, true oder false haben!
-   `function_name` Der Name vom Objekt-State welches unter `enum.function` angelegt wird.
-   `userdata_object_name` Der Name vom Objekt (channel) welches unter `0_userdata.0` angelegt wird. Hier wird der Table_Json für VIS oder die APP gespeichert und aktualisiert.
-   `alias_object_name` Der Name vom Objekt welches unter `alias.0` (channel) angelegt wird. Hier werden dann die States aus der <door_states> neu erstellt und überwacht.
-   `check_alias_object` Bei true wird jeden Tag um 14 Uhr die States unter <alias_object_name> gegen <door_states> geprüft. Existieren dort States die nicht in <door_states> eingetragen sind, erscheint dieses als Warung im Log. Mit der Einstellung <auto_delete_object> können diese automatisch gelöscht werden.
-   `auto_delete_object` Automatische Löschung mit true erlauben.
-   `debuggen` Gibt Logs vom Script aus
-   `emoji_open` Wird ausgegeben wenn ein Fenster geöffnet ist.
-   `emoji_closed` Wird ausgegeben wenn ein Fenster geschlossen ist.
